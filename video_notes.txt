Django Task Management Project Video Notes

Intro:
Hi Professor, this is my Django task-management project. I built it as a simple admin-based application with four main models: Organization, Project, TaskStatus, and Task. The goal was to create a clean design that shows model relationships clearly and demonstrates the basic functionality through the Django administrative portal.

Design approach:
I started by thinking about the real-world structure of a task-management system. At the top level, I created an Organization model because projects usually belong to a company, department, or team. Under that, I created a Project model so each organization can have multiple projects. I then created a TaskStatus model to store reusable workflow states like To Do, In Progress, and Completed. Finally, I created a Task model that ties everything together. Each task belongs to one project and one status, and it can also be assigned to a Django user.

Why the relationships make sense:
The Organization-to-Project relationship is one-to-many because one organization can manage many projects. The Project-to-Task relationship is also one-to-many because one project can contain many tasks. The Task-to-TaskStatus relationship lets the same statuses be reused across many tasks. I also connected tasks to Django's built-in User model so I could assign work to users without building a separate custom user system.

Admin demonstration:
In the admin portal, I can add organizations, projects, statuses, and tasks. I can show that a project is linked to an organization, and each task is linked to both a project and a status. I also configured the admin views with list displays, search fields, filters, and autocomplete fields so the records are easier to manage and demonstrate.

Example walkthrough:
First, I add an organization. Next, I create a project under that organization. After that, I create a few statuses such as To Do, In Progress, and Completed. Then I create tasks and connect each one to a project and a status. I can also assign the task to a user, set the priority, and choose whether the task is completed. This demonstrates the core functionality of the system.

Closing:
Overall, this project shows how Django models can be used to represent a realistic task-management structure and how the Django admin portal can provide basic functionality without needing a custom frontend.
